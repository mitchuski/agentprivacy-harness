The problem

Self-evaluation is a mirage generator

As AI systems increasingly generate work and also judge it — reward models scoring their own rollouts, an LLM grading model output, an agent writing code and then its own tests — one failure recurs. An agent that proposes a change and then checks its own change will pass: not because it cheats, but because the check it invents is the check its change was built to survive.

This is Goodhart's law wearing an agent costume — a problem of self-reference, not of any one domain. The result is what this harness names a mirage: it passes the author's probe and fails the real gate. Mirages are the failure behind reward hacking, behind a judge's documented preference for its own style, behind a test suite that is green because it was written to be green. Wherever a generator can influence the test that judges it, the test stops measuring the world and starts measuring the generator's model of the test.

◿

The answer

Route the information away — don't ask for honesty

Two agents do the work: soulbae 🧙 proposes, soulbis ⚔️ proves. Between them sits the Gap ⿻, whose only job is to derive the verification witnesses by hashing the proposal's own bytes. The proposer chooses its own exam without ever seeing the syllabus; any revision re-seeds the draw.

I(YS ; YM | X) = 0

Given the target, what the prover produces carries no information about what the proposer produces. Not a promise the agents make each other — a property of how the information is routed: the witnesses are a deterministic function of the proposal, so the proposer provably could not have tuned to them. Then a multiplicative gate refuses partial passes (7/8 is a zero, not 87.5%), honest claim-tiers keep results from drifting from evidence, and the door keeps every outward action a human's.

The honest boundary

The limits this evidence does not cross

Stated as plainly as the claims — because the harness would refuse them otherwise.

Prompt-based separation is topology plus discipline, not a proof.

The engine guarantees what it does not put in a seat's prompt, but not what a seat reads from disk. The invariant holds by construction for the routed information; the rest is enforced by explicit per-seat reads and an audit trail where a seat that read past its card is a named failure. Where a domain permits harder separation — separate processes, models, machines, keys — it should be used.

The Gap is not always worth its cost.

A held-out adversarial draw earns its price only when the claim space is too large to check. Where every claim is enumerable, an exhaustive auditor gives certainty for free. The repo keeps a retired instance precisely to record where the harness was applied and should not have been. A harness is for adversaries; an auditor is for facts.

This is a runnable demonstration, not a controlled study.

The evidence is a reproducible repository and a worked frontier, not a benchmarked comparison against baselines under peer review. The claim is that the construction is coherent, checked, and demonstrably catches a class of failure on the instances shown — not that it is optimal.

A proposer that grades its own work builds mirages; only what the Gap could not tune to is a result.
